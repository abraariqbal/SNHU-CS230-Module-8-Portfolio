Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

The client for this project was The Gaming Room a company that wanted to expand its Android game, Draw It or Lose It, to multiple different operating platforms.
The goal was to design a web-based application that could support desktop and mobile users while allowing multiple game sessions to run at the same time. The software also
needed to be scalable, secure, and be able to communicate across different operating systems using a client-server architecture.

What did you do particularly well in developing this documentation?

I think I did well evaluating different operating platforms and explaining what solutions would be best for the client.
Throughout the documentation, I was able to connect my recommendations to The Gaming Room's requirements like security and multi-platform support.
This helped make sure that the decisions were aligned with the client's goals and needs.

What about the process of working through a design document did you find helpful when developing the code?

Working through the design document gave me a clear understanding of what the application needed to do and how the different components would work together.
Having the requirements, design decisions, and architecture planned out made it easy to implement the code and stay organized throughout the development.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could revise one part of my work I would improve my discussion of operating system architectures. I could have provided more detail about 
how the architecture of recommended platform supports the performance and scalability of the application. Expanding on that section would have made a 
stronger connection between Linux's architecture and the needs of Draw It or Lose It.

How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?

I interpreted the client's needs by focusing on their goal of expanding Draw It or Lose It into a secure, scalble, multi platform application. Since it 
was my job to turn the clients ideas/requirements into a software design, I made recommendations that supported their requirements like using Linux for the server,
having a client-server architecture, and role based security access control. Considering the users needs is important because the software should solve the clients problem.

How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

I approached designing the software by understanding the client’s requirements and then evaluating looking at different solutions before making recommendations.
Breaking the project into sections made the design process manageable and organized. In the future, I would continue using this approach by gathering requirements first,
comparing different design options, and creating a detailed design document before writing code. This helps ensure the final software meets the client’s needs.
